# 网络安全技术基础综合实验 — 需求分析文档

## 1. 项目概览

### 1.1 核心目标
开发一套完整的网络传输安全演示系统，包含发送端与接收端，实现以下安全能力：
- 传感器数据的**采集与封装**
- 对称加密算法（AES）的**自主实现**，用于加密原始数据
- 公钥密码算法（RSA）的**自主实现**，用于加密对称密钥
- 消息认证码（HMAC）或数字签名机制的**自主实现**，用于数据完整性认证
- 基于 Socket 的**网络传输**，实现加密认证数据的发送与接收
- 接收端对数据进行**验证、解密、还原**

### 1.2 约束条件
- 加密算法和认证算法必须**自行编写代码实现**，不得调用 OpenSSL、Cryptography 等现成加密库
- 必须提供可独立运行的源代码
- 编程语言推荐：Python 3（兼顾可读性与实现难度）

### 1.3 目标用户
- 网络安全课程实验教学与评估
- 教师验收重点：算法自主实现、流程完整性、代码可读性

## 2. 功能需求

### 2.1 模块划分

| 模块 | 说明 |
|------|------|
| 数据采集模块 | 模拟传感器数据生成或从文件/网络捕获原始数据 |
| 对称加密模块 | 自实现 AES-128 算法，对原始数据进行加密 |
| 公钥加密模块 | 自实现 RSA 算法，对 AES 密钥进行加密 |
| 消息认证模块 | 自实现 HMAC-SHA256 或数字签名，对密文进行认证 |
| 网络传输模块 | Socket 编程实现发送端 → 接收端的数据传输 |
| 接收验证模块 | 接收数据、验证认证码、解密还原原始数据 |

### 2.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1 | 传感器数据模拟生成 | 生成温度、湿度等模拟传感器数据（JSON 格式） | Must |
| F2 | AES-128 加解密 | 自实现 AES-128 ECB/CBC 模式，支持 PKCS7 填充 | Must |
| F3 | RSA 密钥生成 | 自实现 RSA 密钥对生成（512/1024 bit） | Must |
| F4 | RSA 加密/解密 | 使用 RSA 公钥加密 AES 密钥，私钥解密 | Must |
| F5 | HMAC-SHA256 消息认证码 | 自实现 HMAC 算法，对密文计算认证码 | Should |
| F6 | 数据封装（发送端） | 封装为"密文 + 加密后的密钥 + 认证码"的传输格式 | Must |
| F7 | Socket 发送 | 发送端通过 TCP Socket 发送封装后的数据 | Must |
| F8 | Socket 接收 | 接收端监听端口，接收完整数据包 | Must |
| F9 | 完整性验证 | 接收端重新计算 HMAC 并比对，验证数据完整性 | Must |
| F10 | 数据解密还原 | 使用 RSA 私钥解密 AES 密钥，再用 AES 解密数据 | Must |
| F11 | 实验结果输出 | 打印/展示原始数据、加密数据、解密后数据的对比 | Should |

### 2.3 数据处理流程

```
发送端：
  原始传感器数据
      ↓
  [AES 加密]  ←  AES 密钥（随机生成）
      ↓
  数据密文 ──────────────────────────────┐
      ↓                                  │
  [HMAC 认证] ←  HMAC 密钥              │
      ↓                                  │
  数据密文 + HMAC 值  ───────────────┐   │
                                      ↓   ↓
  [RSA 加密] ← RSA 公钥         加密的 AES 密钥 + 加密的 HMAC 密钥
      ↓
  最终传输包 = [数据密文] + [RSA 加密后的 AES 密钥] + [RSA 加密后的 HMAC 密钥] + [HMAC 值]

接收端：
  接收到传输包
      ↓
  用 RSA 私钥解密 → 得到 AES 密钥 + HMAC 密钥
      ↓
  用 HMAC 密钥重新计算 HMAC → 与收到的 HMAC 值比对
      ↓  (验证通过)
  用 AES 密钥解密密文 → 还原原始传感器数据
```

## 3. 接口定义

### 3.1 内部函数接口（Python 模块级）

#### 3.1.1 AES 模块（`aes.py`）

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `aes_encrypt(plaintext, key)` | plaintext: bytes, key: bytes(16) | bytes | AES-128 加密 |
| `aes_decrypt(ciphertext, key)` | ciphertext: bytes, key: bytes(16) | bytes | AES-128 解密 |
| `key_expansion(key)` | key: bytes(16) | list[word] | 密钥扩展 |
| `add_round_key(state, round_key)` | state: 4x4 matrix, round_key | 4x4 matrix | 轮密钥加 |
| `sub_bytes(state)` | state: 4x4 matrix | 4x4 matrix | S 盒字节代换 |
| `shift_rows(state)` | state: 4x4 matrix | 4x4 matrix | 行移位 |
| `mix_columns(state)` | state: 4x4 matrix | 4x4 matrix | 列混合 |

#### 3.1.2 RSA 模块（`rsa.py`）

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `generate_keypair(bits)` | bits: int | (public_key, private_key) | RSA 密钥对生成 |
| `rsa_encrypt(message, public_key)` | message: int, public_key | int | RSA 加密 |
| `rsa_decrypt(ciphertext, private_key)` | ciphertext: int, private_key | int | RSA 解密 |
| `gcd(a, b)` | a, b: int | int | 最大公约数 |
| `mod_inverse(a, m)` | a, m: int | int | 模逆元 |
| `is_prime(n)` | n: int | bool | 素数判定（Miller-Rabin） |

#### 3.1.3 HMAC 模块（`hmac_.py`）

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `hmac_sha256(key, message)` | key: bytes, message: bytes | bytes | HMAC-SHA256 计算 |
| `sha256(message)` | message: bytes | bytes(32) | SHA-256 哈希（自实现） |

#### 3.1.4 网络传输模块（`network.py`）

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `send_data(host, port, data_packet)` | host: str, port: int, data_packet: bytes | bool | 发送数据包 |
| `receive_data(host, port)` | host: str, port: int | bytes | 接收数据包 |

#### 3.1.5 主控模块（`main.py` / `sender.py` + `receiver.py`）

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `sender_main()` | 无 | None | 发送端主流程 |
| `receiver_main()` | 无 | None | 接收端主流程 |

### 3.2 传输协议格式（自定义）

```
数据包结构（字节流）:
┌─────────────────────────────────────────────────────┐
│ 字段 1: 密文长度 (4 bytes, big-endian uint32)       │
│ 字段 2: 密文数据 (length = 密文长度)                 │
│ 字段 3: RSA 加密后的 AES 密钥长度 (4 bytes)          │
│ 字段 4: RSA 加密后的 AES 密钥数据                    │
│ 字段 5: RSA 加密后的 HMAC 密钥长度 (4 bytes)          │
│ 字段 6: RSA 加密后的 HMAC 密钥数据                    │
│ 字段 7: HMAC 值 (32 bytes)                           │
└─────────────────────────────────────────────────────┘
```

## 4. 数据库模型

本实验不涉及数据库持久化存储。所有数据在运行时存在于内存中，流程如下：

| 数据项 | 产生位置 | 消费位置 | 说明 |
|--------|----------|----------|------|
| 原始传感器数据 | `sender.py` | 用于加密展示 | 模拟的温度/湿度等数值 |
| AES 密钥 (16 bytes) | `sender.py` | AES 加解密 + RSA 加密传输 | 每次发送随机生成 |
| HMAC 密钥 (32 bytes) | `sender.py` | HMAC 计算 + RSA 加密传输 | 每次发送随机生成 |
| RSA 密钥对 | `sender.py` 生成 | 发送端加密 / 接收端解密 | 仅程序启动时生成一次 |
| 密文 | `sender.py` AES 加密 | 网络传输 + 接收端解密 | — |
| HMAC 值 | `sender.py` 计算 | 网络传输 + 接收端验证 | — |

## 7. 开发任务拆分（MoSCoW 优先级）

### Must Have（必须完成）
1. **AES-128 算法自实现**（含 S 盒、密钥扩展、加解密流程）
2. **RSA 算法自实现**（含密钥生成、加解密、Miller-Rabin 素性检测）
3. **HMAC-SHA256 自实现**（含 SHA-256 哈希自实现）
4. **Socket 网络传输**（TCP 发送与接收）
5. **发送端主程序**（数据生成 → AES 加密 → HMAC 认证 → RSA 加密密钥 → 发送）
6. **接收端主程序**（接收 → RSA 解密密钥 → HMAC 验证 → AES 解密 → 还原）
7. **控制台日志输出**（每一步的中间结果展示）

### Should Have（建议完成）
8. **AES-CBC 模式**（替代 ECB，提供更高安全性）
9. **数据包长度头设计**（解决 TCP 粘包问题）
10. **错误处理与异常提示**（密钥不匹配、认证失败等场景）

### Could Have（条件允许时完成）
11. **GUI 界面**（Tkinter 简单界面）
12. **真实传感器数据采集**（如从串口读取）
13. **多线程并发处理**（同时处理多个连接）

## 8. 技术选型建议

| 层面 | 选择 | 理由 |
|------|------|------|
| 编程语言 | Python 3 | 开发速度快，大整数运算原生支持（RSA 需要），字节操作便捷 |
| 网络模型 | TCP Socket (socket 标准库) | 可靠传输，适合实验场景 |
| 对称加密 | AES-128 | 算法规范清晰，S 盒固定，适合自实现 |
| 公钥加密 | RSA (512-1024 bit) | 数学原理清晰，适合教学演示 |
| 哈希算法 | SHA-256 | HMAC 标准算法，安全性足够 |
| 密钥管理 | RSA 加密传输对称密钥 | 解决对称密钥分发问题 |

## 9. 风险和注意事项

| 风险 | 应对方案 |
|------|----------|
| AES S 盒实现错误导致加解密失败 | 对照 NIST FIPS-197 标准逐字节验证 |
| RSA 大素数生成过慢 | 使用 Miller-Rabin 概率检测，bit 数控制在 512~1024 |
| TCP 粘包导致数据解析错误 | 自定义协议头（长度字段），接收端按长度读取 |
| HMAC 密钥与 AES 密钥混淆 | 代码中明确变量命名，加注释区分 |
| 自实现 SHA-256 复杂度高 | 可考虑先实现简化版本，确保 HMAC 流程正确 |
